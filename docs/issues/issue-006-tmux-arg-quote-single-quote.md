# [低] 参数包含单引号时透传失败

## 概要
`codex_cmd` 通过拼接 `'$arg'` 构造，未转义单引号，导致含 `'` 的参数在 tmux 命令行中断或被截断。

## 影响
- 启动命令参数不可靠。
- 复杂提示词或 shell 参数无法正确传递。

## 位置
- `bin/codex-hud:313`
- `bin/codex-hud:318`

## 复现步骤
1. 执行 `codex-hud "test 'quote'"`。
2. 观察 codex 实际接收参数。

## 预期结果
参数完整传递。

## 实际结果
参数被截断或语法错误。

## 修复建议
- 使用安全的 shell 转义（如 `printf %q`）拼接。
- 或改为通过 tmux `send-keys` 传参数组形式。
