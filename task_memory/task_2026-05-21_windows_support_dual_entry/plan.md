# Task Plan: Windows Dual Entry Support

## Goal

Complete `feature/windows-support-dual-entry` so Codex HUD works smoothly on Windows WSL first-class explicit mode, and also supports native PowerShell and cmd entrypoints.

## Confirmed Decisions

- Default `codex` entry is Native-first: native PowerShell HUD, WSL HUD fallback, then plain native Codex CLI.
- Explicit `codex-hud-wsl` is the high-reliability full-HUD path for WSL Ubuntu.
- WSL provisioning uses root or sudo when available, and fails fast with exact manual commands when permissions are insufficient.
- Tracked `node_modules/.bin/tsc` and `node_modules/.bin/tsserver` are removed from version control because platform-specific shims must be generated by `npm install`.

## Implementation Phases

- [x] Phase 1: Persist plan and baseline findings in `task_memory/`.
- [x] Phase 2: Fix Windows build root cause and verify `npm.cmd run build`.
- [x] Phase 3: Add failing regression tests for WSL first-run, WSL wrapper temp files and quoting, WSL provisioning, cmd shims, and runtime state precedence.
- [x] Phase 4: Implement the minimal code changes to pass the regression tests.
- [x] Phase 5: Update README files and write the final Markdown test report.
- [x] Phase 6: Run all available verification commands and document blocked environment checks.

## Acceptance Criteria

- `npm.cmd run build` exits 0 on Windows.
- Unit tests cover all changed TypeScript code paths and pass.
- Windows PowerShell harness covers Native-first fallback, explicit WSL wrapper, installer profile management, cmd shim creation, and uninstall cleanup.
- WSL wrapper does not write temporary executable files into the repo checkout.
- Missing default Codex session directories do not crash HUD startup; explicitly invalid override paths still fail fast.
- Runtime pane state can override stale rollout/config UI state for mode, approval policy, and sandbox mode.
- README English and Chinese Windows support sections match the implemented behavior.
- Test report records commands, environment, pass/fail status, and numeric metrics.

## Status

Available verification is complete. Real WSL distro and Bash/tmux integration checks remain environment-blocked on this host and are documented in the test report.
